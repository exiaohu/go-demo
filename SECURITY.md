## Reporting a Vulnerability

Use this section to tell people how to report a vulnerability.

Tell them where to go, how often they can expect to get an update on a
reported vulnerability, what to expect if the vulnerability is accepted or
declined, etc.

If you believe you have found a security vulnerability, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

### Reporting Process

1.  Email `security@example.com` with the details of the vulnerability.
2.  We will acknowledge your report within 48 hours.
3.  We will investigate the issue and provide a timeline for a fix.
4.  Once fixed, we will release a security patch and credit you (if desired).
